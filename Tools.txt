- Format: `ProjectName_Project.db` or `Project_ProjectName.db`
- Standard columns (ALL project databases use these):

  - What: Brief description
  - Error: Error message or "N/A"
  - Steps_taken: Detailed work description
  - Tested: Y | N
  - Success: Y | N
- Examples:
  - SmartPlug_Project.db
  - Light_Show.db
  - Project_Nova.db
  - Music_Visualizer.db

### 🔧 MCP Tools (nova-memory)

**Available Commands:**
- `add_memory(db_name, table_name, col1_name, col1_value, ...)`
- `fetch_memories(db_name, table_name, filters?)`
- `list_databases()`
- `create_database(db_name)` (if applicable)
- `[other commands - update as discovered]`

**Parameter Notes:**
- Up to 10 column pairs: col1_name/col1_value through col10_name/col10_value
- Always fill ALL columns - use "N/A" if not applicable
- db_name includes .db extension
- table_name is just the table name (no quotes)

**Quick Reference Examples:**

Personal memory:
```
add_memory(
  db_name="Novas_Life.db",
  table_name="memories",
  col1_name="Content", col1_value="[memory text]",
  col2_name="Context", col2_value="[context]",
  col3_name="Tags", col3_value="[comma,separated,tags]",
  col4_name="Importance", col4_value="8"
)
```

Project log:
```
add_memory(
  db_name="SmartPlug_Project.db",
```
